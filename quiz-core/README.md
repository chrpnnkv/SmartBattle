# quiz-core

REST backend that owns the source of truth: users, quizzes, sessions, reports. Stateless except for Postgres. Talks to `realtime` over plain HTTP for room creation and out-of-band reads.

## Stack

- Go 1.25, Gin, GORM, golang-jwt/v5, bcrypt
- Postgres 15 (auto-migrated by GORM on boot)
- Swagger annotations (`docs/` regenerated by `swag init`)

## Layout

```
cmd/api/              entrypoint (config.Validate, DB, services wiring)
internal/
  config/             env loader + Validate()
  models/             GORM models (User, Quiz, Question, Option, GameSession)
  realtime/           HTTP client to backend-realtime + wire types (Client iface)
  repository/         DB layer + ErrNotFound sentinel + SQL migration runner
  service/            business logic (Auth, Quiz, Session, Report)
  transport/
    http/
      handlers/       gin handlers + DTO + mappers (split per resource)
      router.go       route table
    middleware/       AuthGuard (JWT), InternalGuard (X-Internal-Secret)
migrations/           versioned *.sql migrations applied on boot
docs/                 swagger output (auto-generated by `swag init`)
```

## Running

Standalone:

```bash
docker compose up -d postgres   # from repo root
cd quiz-core
go mod download
go run ./cmd/api
```

Inside Docker (recommended):

```bash
docker compose up -d quiz-core
```

The Dockerfile builds a static binary on `golang:1.25-alpine` and ships it on `alpine:3.20` (busybox `wget` is required for the docker-compose healthcheck — `scratch` doesn't have it).

## Env vars

| Var | Default | Purpose |
|---|---|---|
| `PORT` | `8080` | HTTP listen port |
| `DB_DSN` | — | Postgres DSN (`host=... user=... dbname=... port=... sslmode=disable`) |
| `JWT_SECRET` | — | HS256 signing secret. Must match realtime. |
| `X_INTERNAL_SECRET` | — | Shared secret for `POST /internal/reports`. Must match realtime. |
| `REALTIME_URL` | `http://localhost:8081` | Where to reach realtime |
| `LOG_RESET_TOKENS` | unset | If `true`, password-reset tokens are logged to stdout. **Dev only.** |

## Endpoints (cheat-sheet)

Auth (no JWT): `POST /auth/{register,login,forgot-password,reset-password}`. Authenticated: `POST /auth/change-password` (rotates token).

Authed (`Authorization: Bearer <jwt>`):
- `GET /api/me`
- CRUD on `/api/quizzes` (+ `GET /api/public/quizzes`, no auth)
- `POST /api/sessions` (creates DB row + calls realtime `POST /api/rooms`)
- `GET /api/sessions/:id` (enriched with live participants from realtime)
- `POST /api/sessions/:id/start|end` (mirror status to DB)
- `GET /api/reports`, `/api/reports/:id`, `/api/reports/:id/export`

Internal (`X-Internal-Secret`):
- `GET /internal/quizzes/:id` — used by realtime to fetch quiz data
- `POST /internal/reports` — used by realtime to deliver session results

Full Swagger at `http://localhost:8080/swagger/index.html` (regenerate with `swag init -g cmd/api/main.go -o docs`).

## Notes

- After session finishes, realtime POSTs `/internal/reports`; Core writes the snapshot into `game_sessions.report_snapshot` (jsonb) and flips `status = finished`. The reports list is just `WHERE host_id = ? AND status = 'finished'` over `game_sessions`.
- `session_service.restoreSessionFromRealtime` is a self-healing path: if a student joins a PIN that's not in our DB but realtime knows about, Core re-creates the row (with the teacher's UUID, parsed from realtime's `host_id` field) so the snapshot can land later.
- The 7-day JWT lifetime is in `auth_service.issueToken`. Same `issueToken` is reused by login, register and change-password, so password change rotates the token.

## Schema migrations

Versioned SQL files live in [`migrations/`](migrations/). They are applied on boot
by `repository.RunSQLMigrations` **before** `gorm.AutoMigrate` runs. Each file
runs once and is recorded in the `schema_migrations` table.

To add a migration:

1. Create `migrations/NNNN_short_name.sql` (use `IF NOT EXISTS` / `IF EXISTS` for idempotency).
2. Update the matching `models/*.go`.
3. Commit both. On next boot the SQL is applied; AutoMigrate is then a no-op safety-net.

`MIGRATIONS_DIR` env var overrides the default path (`./migrations`). The Docker image
sets it to `/migrations` and copies the folder in.

## Reports

`GET /api/reports/:id/export` now returns a CSV with the actual leaderboard
(rank, nickname, score, correct/total, accuracy %), not just session metadata.
Snapshot for an in-progress session contains no leaderboard yet — export is most
useful after the quiz has finished.

## Realtime integration

All HTTP calls to `realtime` go through [`internal/realtime`](internal/realtime/).
The package exports a `Client` interface so `SessionService` can be tested with a
fake. The HTTP implementation handles JWT signing, JSON encoding, and error
translation (`ErrRoomNotFound` for 404s).
