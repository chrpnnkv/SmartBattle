# Migrations

Файлы в этом каталоге описывают каноническую схему БД. Имя — `NNNN_<описание>.sql`,
где `NNNN` — порядковый номер. Файлы выполняются по возрастанию имени, ровно один раз
каждый, что отслеживается в служебной таблице `schema_migrations`.

Запуск — автоматически на старте сервера (`internal/repository/migrations.go`),
**до** `gorm.AutoMigrate`. AutoMigrate оставлен как safety-net для разработки —
он добавит то, что забыли вписать в SQL, но в production он становится no-op,
если все колонки уже созданы.

## Как добавить миграцию

1. Создать новый файл, например `0002_add_question_image_url.sql`.
2. Написать только идемпотентные `IF NOT EXISTS` / `IF EXISTS` SQL.
3. Закоммитить — и саму миграцию, и соответствующую правку моделей в `internal/models/`.
4. На любом запуске, на котором эта миграция ещё не применялась, она выполнится один раз
   и запишется в `schema_migrations`.

## Как откатить

Down-миграции не реализованы намеренно: проще написать новый up-файл с обратным
SQL, чем пытаться корректно откатить через GORM. Это уровень "хорошее
универститетское задание" — для production стоит брать `golang-migrate` или `goose`.
